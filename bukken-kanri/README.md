# 🏠 物件管理アプリ（MVP）

土地情報・収益物件情報を「登録／一覧／編集／削除／検索」できる、
Next.js 製のシンプルなWebアプリのMVP（試作品）です。

---

## 1. 必要なもの

このアプリを動かすには **Node.js** が必要です。

- まだ入っていない場合：https://nodejs.org/ から「LTS版」をダウンロードして
  インストールしてください（インストーラの指示に従うだけでOK）。
- 入っているか確認するには、ターミナル（黒い画面）で次を実行：

```bash
node -v
```

`v20.x.x` のように数字が出ればOKです。

---

## 2. 起動方法（3ステップ）

ターミナルを開き、このフォルダ（`bukken-kanri`）に移動してから、
上から順にコマンドを実行してください。

### ① フォルダに移動

```bash
cd bukken-kanri
```

### ② 必要な部品をインストール（初回だけ）

```bash
npm install
```

※ 少し時間がかかります。`node_modules` というフォルダが作られたら成功です。

### ③ アプリを起動

```bash
npm run dev
```

成功すると、ターミナルに次のような表示が出ます：

```
▲ Next.js 14.2.5
- Local: http://localhost:3000
```

ブラウザで **http://localhost:3000** を開くとアプリが表示されます。
（スマホの見え方を確認したいときは、ブラウザの検証ツールで
スマホ表示に切り替えるか、後述の方法で実機からアクセスできます）

> 止めたいときは、ターミナルで `Ctrl + C` を押します。

---

## 3. スマホ実機で確認したいとき（任意）

PCとスマホを **同じWi-Fi** につないだ状態で：

```bash
npm run dev
```

を実行し、PCのIPアドレス（例：192.168.0.5）を調べて、
スマホのブラウザで `http://192.168.0.5:3000` を開きます。

---

## 4. 使い方

- 右上の **「＋ 新規登録」** で物件を追加します。
- 「土地情報」「収益物件情報」をタブで切り替えて入力します。
- 入力すると **坪単価・一種単価・専有単価などが自動計算** されて、
  画面の「📊 自動計算」欄にすぐ表示されます。
- トップ画面では **タブ（すべて／土地／収益物件）** と **検索ボックス** で
  絞り込みできます。
- 各カードの **「編集」「削除」** から修正・削除ができます。

### 自動計算の式

| 項目 | 計算式 |
| --- | --- |
| 地積坪 | 地積㎡ ÷ 3.305785 |
| 坪単価 | 価格 ÷ 地積坪 |
| 一種単価 | 価格 ÷ 地積坪 ÷ 容積率 × 100 |
| 専有単価 税別 | 価格 ÷ 貸床面積 |
| 専有単価 税込 | 専有単価 税別 × 1.1 |

> ※ 価格の単位は「万円」で入力する前提です。

## 5. データの保存について（重要）

このMVPでは、データは **使っているブラウザの中（localStorage）** に
保存されます。手軽ですが、次の制限があります：

- 別のPC・スマホとはデータが共有されません。
- ブラウザのデータを消すと物件データも消えます。

本格運用では、次のステップで **Supabase**（クラウドのデータベース）に
移行します。

---

## 6. あとでSupabaseに移行するには

このアプリは移行しやすいように作ってあります。
**`lib/storage.ts` というファイルの中身だけ** を書き換えればOKで、
画面側のファイルは一切変更不要です。

おおまかな手順：

1. Supabaseでプロジェクトを作成し、`properties` テーブルを用意
   （列は `lib/types.ts` の項目と同じにする）
2. `npm install @supabase/supabase-js`
3. `lib/storage.ts` の `getAll / getById / create / update / remove` の
   中身を Supabase の呼び出しに書き換える
   （ファイル末尾にサンプルコードをコメントで入れてあります）

## 7. フォルダ構成

```
bukken-kanri/
├── app/
│   ├── layout.tsx              共通の外枠（ヘッダー）
│   ├── globals.css             デザイン（スマホ優先）
│   ├── page.tsx                トップ＝一覧・検索・削除
│   └── properties/
│       ├── new/page.tsx        新規登録ページ
│       └── [id]/edit/page.tsx  編集ページ
├── components/
│   └── PropertyForm.tsx        登録・編集の共通フォーム
├── lib/
│   ├── types.ts                データの型・選択肢の定義
│   ├── calc.ts                 自動計算のロジック
│   └── storage.ts              データ保存（★ここを変えればSupabase移行）
├── package.json                コマンドと依存パッケージ
└── README.md                   このファイル
```

困ったときは、まず `npm install` からやり直すと直ることが多いです。
